# Portfolio — React + Vite (Pinned Toolchain)

(Persian). The goal is to practice layout, typography, and component composition, then enhance the UI with a slideshow (carousel) using either a lightweight UI library (e.g., Bootstrap or Material) or a custom solution. The app includes basic routing (Home, About, Projects, Contact) and simple components like a header, footer, and a contact form with validation.

## 0) Prerequisites

- Node.js: `>=22.12.0 <23` **or** `^20.19.0`
- npm: recommended `11.6.2`

Check your versions:

    node -v
    npm -v

## 1) Init a local Git repository

    mkdir emoji-search-react-vite
    cd emoji-search-react-vite
    git init

## 2) Choose versions (query npm)
We’ll pin these three:

    npm view create-vite version          # => X1 (e.g. 8.0.2)
    npm view vite version                 # => X2 (e.g. 7.2.2)
    npm view @vitejs/plugin-react version # => X3 (e.g. 5.1.0)


Check Node engine range for Vite (for your engines.node):

    npm view vite@7.2.2 engines
    # Example: { node: '^20.19.0 || >=22.12.0' }


In this repo we used:
- create-vite@8.0.2,
- vite@7.2.2,
- @vitejs/plugin-react@5.1.0,
- npm@11.6.2,
- engines.node=">=22.12.0 <23 || ^20.19.0".


## 3) Scaffold with the scaffolder’s version pinned
The tag after create@ pins the scaffolder only. We’ll pin the runtime deps next.

    npm create vite@8.0.2 . -- --template react
    # If it asks about using the current non-empty dir (because of .git), choose Yes.


## 4) Install and pin exact dependency versions

    # Save exact versions (no ^ or ~) in this project
    npm config set save-exact true --location=project

    # Install deps from the generated package.json
    npm i

    # Pin Vite and the React plugin explicitly
    npm pkg set "devDependencies.vite=7.2.2" "devDependencies.@vitejs/plugin-react=5.1.0"

    # Align the lockfile after edits
    npm i


## 5) Pin the toolchain & enforce Node compatibility

    # Get your npm version (e.g. 11.6.2)
    npm -v

    # Pin package manager and Node engine range
    npm pkg set packageManager="npm@11.6.2" "engines.node=>=22.12.0 <23 || ^20.19.0"

    # (Optional) Enforce engines during install (CI/teammates get a hard error if incompatible)
    npm config set engine-strict true --location=project


## 6) .gitignore — What/Why/When/How
What is it?
A file that tells Git which paths to ignore (don’t track/commit).

Why do we need it?
To prevent committing large or sensitive/ephemeral files (e.g., node_modules, build outputs, editor settings, secrets).

When should I add it?
Right after git init and before your first commit.
If you forgot, add it now; anything already tracked will stay tracked until you remove it from Git history.

How to add it (minimal Node/Vite setup):
- add these four line in .gitignore :
node_modules
dist
.vscode
.env


- or use this command to add them to .gitignore:

    @'
    node_modules
    dist
    .vscode
    .env
    '@ | Add-Content -Encoding utf8 .gitignore


show .gitignore content quickly:

    Get-Content .gitignore


- node_modules: installed packages (large, regenerated by npm i).

- dist: production build output.

- .vscode: personal editor settings (avoid sharing noise).

- .env: environment variables/secrets you don’t want in Git.
